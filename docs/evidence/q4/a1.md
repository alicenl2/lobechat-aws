# TL;DR

- **Cost:** at our 200-user scale, self-hosting runs **~$16/user/month all-in (~$115k over 3 years)** vs **$60/user ($432k) for ChatGPT Enterprise** and **$30/user ($216k) for Microsoft Copilot** — a **~$317k (73%) saving** over the Enterprise option, even after loading in a part-time DevOps owner.
- **Compliance is the real driver, not price:** our client tenders, CVs and financials are **NDA-bound**; several client contracts forbid third-party processing. Self-hosting keeps 100% of that data in our own `eu-west-1` VPC. A SaaS DPA does not clear that bar.
- **Recommendation:** **self-host**, conditional on funding ≥0.25 FTE platform ownership and accepting no vendor uptime SLA. If we cannot staff the ops, **buy Microsoft Copilot** (cheapest SaaS, already in our Microsoft estate).

# Lock-in analysis

Switching cost by dimension, with each option's exit cost:

**Data.** Self-hosted, company data stays in **our own AWS environment** — S3, RDS,
EC2/EBS in `eu-west-1`. If we stop using the platform, the documents, metadata, vector
database and logs are already under our control in standard cloud services, so the
**exit cost is low**. With **ChatGPT Enterprise**, uploaded documents and conversation
history live inside OpenAI's platform; export is possible, but we'd still need engineering
work to clean, transform and re-import the data elsewhere — **medium**. With **Microsoft
Copilot**, the tool is deeply tied to Microsoft 365 (Teams, SharePoint, OneDrive, Entra
ID): a strength if we're already Microsoft-native, but a **high exit cost** because the AI
experience becomes bound to the whole Microsoft environment.

**Identity.** Self-hosting gives more control over authentication: **Casdoor** connects to
an existing identity provider over standard SSO (OIDC / SAML), so if we later change IdP
the application layer adapts without replacing the whole AI system. **ChatGPT Enterprise**
supports SSO, but OpenAI still controls the user workspace and session experience.
**Copilot** is even more tied to **Microsoft Entra ID** — convenient, but a dependency on
Microsoft identity infrastructure.

**Model.** A self-hosted LobeChat can route each task to a different model — Claude Haiku
for fast extraction, Claude Sonnet for high-quality drafting, **Mistral** for French-language
tasks, or a **local model** for sensitive use cases — so we never depend on one provider.
**ChatGPT Enterprise** mainly locks us into OpenAI models; **Copilot** is tied to Microsoft's
stack and Azure OpenAI. Both are powerful but offer less flexibility the day another model
becomes cheaper, faster, more compliant, or simply better for a specific internal workflow.

**Tooling.** The biggest advantage. **MCPHub** lets the platform connect to custom tools —
internal CV databases, proposal templates, document parsers, OCR, Qdrant RAG collections,
AWS tools, Notion, file systems, custom APIs. SaaS products are more polished but far less
open to deep workflow customisation: Copilot is excellent *inside* Microsoft 365 and ChatGPT
Enterprise is strong for general assistance, but neither gives the same control over custom
MCP tools and company-specific pipelines.

**Net:** self-host keeps exit cost low on all four dimensions; SaaS concentrates lock-in in
**model and tooling** (and, for Copilot, data and identity too) — the hardest to unwind once
consultants build daily workflows on top.

# Data sovereignty + compliance

Daveo operates in the **legal / consulting** sector and is bound by client **NDAs**. A
single data breach would be a **contractual and reputational catastrophe** — losing a
tender, or a client, over leaked documents costs far more than any tooling licence.
Self-hosting removes the exposure that comes with handing that data to an external tool.

- **Residency:** with the stack in our own AWS account, sensitive documents **never leave
  the EU (France / Ireland)** — they stay in a VPC we control. That satisfies **GDPR**
  by design.
- **Controllership:** Daveo remains the **data controller**; AWS is only a processor
  acting under our instructions and our region choice. We keep full erasure and access
  control.
- **ChatGPT Enterprise:** OpenAI by default processes data on **US infrastructure**.
  Standard Contractual Clauses exist, but **CNIL has treated US cloud providers as
  higher-risk for sensitive EU data since Schrems II** — a hard sell to a client whose
  NDA forbids third-country processing.
- **Microsoft Copilot:** the **EU Data Boundary** helps, but it does **not guarantee that
  100% of processing stays on EU territory** (some support and telemetry flows still
  cross out).

The decisive advantage: with self-hosting we can hand a client a **data-flow diagram
showing their documents never leave a VPC Daveo controls** — a concrete assurance no
SaaS DPA can match, and one that doubles as a trust signal in the pitch itself.

# Capability gap

**What self-hosting does that SaaS cannot:**

- **Internal tool access.** MCPHub wires LobeChat **directly into our internal systems** — the CV database, the past-proposal store, the ESG document templates. No SaaS product connects to those.
- **Custom RAG over proprietary documents.** Qdrant + Hayhooks let us tune retrieval to **our own domain vocabulary** (tenders, consulting jargon, client names); SaaS RAG is generic and cannot be tuned to it.
- **Model routing.** We pick the right model per task — **Haiku for extraction (cheap), Sonnet for drafting (quality)** — instead of one SaaS model forced on every job.
- **Per-user / per-team budget control.** We can cap token spend by user or team, stopping (for example) a junior consultant from **burning $500 in a single session** — exactly the fair-use guardrail the Q3 pricing model relies on.

**What SaaS does that self-hosting cannot (without building it ourselves):**

- **Uptime SLA.** ChatGPT Enterprise offers a **99.9% SLA**; our self-hosted stack has **no SLA unless we engineer one** (HA, failover, monitoring).
- **Vendor support.** With SaaS there's someone to call; self-hosted, **we are on our own**.
- **Zero DevOps.** SaaS needs no platform team; self-hosted needs the **platform-ops FTE (0.25–0.5)** costed into the TCO above.

# Recommendation

**Self-host — provided all three conditions hold:**

1. **The team can commit ~0.5 FTE to platform maintenance.** Self-hosting is only cheaper than SaaS if we actually own the ops; without a named owner the TCO math (and the uptime) falls apart.
2. **Daveo keeps handling sensitive, NDA-bound material** — client tender documents, CVs, financials — that cannot legally sit with a third-party processor.
3. **Owning the platform has strategic value beyond cost:** (a) it is a **differentiator in client pitches**, a live demo of what Daveo can build; and (b) our work needs **multiple models** (Haiku to extract, Sonnet to draft, freedom to swap providers), so a single-vendor SaaS would mean either paying for several platforms or accepting one locked-in model.

**Otherwise, buy.** If those conditions stop holding — the DevOps team can no longer carry the stack, or we stop handling confidential data — then **ChatGPT Enterprise or Microsoft Copilot** is the better call: we trade the saving for a vendor SLA, support and zero ops.

# Reversal trigger

Any one of these future events flips the recommendation from *build* to *buy*:

- **A SaaS vendor goes verifiably EU-sovereign.** If OpenAI or Microsoft ships an offering with data processed **exclusively on EU soil, auditable and CNIL-approved**, at current Enterprise pricing, our strongest argument — compliance — disappears, and SaaS becomes attractive for its zero ops overhead.
- **We lose DevOps capacity.** If the platform owner departs and the stack can no longer be maintained, then once you **price in the cost of hiring a replacement**, SaaS becomes the cheaper option.
- **Acquisition + tooling standardisation.** If Daveo is absorbed by a larger group with a standardised Microsoft 365 estate (e.g. **Magellan Partners**) that mandates Copilot/M365, corporate standardisation overrides the local build-vs-buy math.
