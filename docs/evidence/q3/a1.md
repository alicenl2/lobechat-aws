# Q3 — Cost model + unit economics

All prices are **AWS list prices for `eu-west-1` (Europe, Ireland)**, the region
used by the Daveo prod design in Q2. Lines marked **(screenshot)** are verified
against the captures in `assets/` (accessed **2026-06-02**); the remaining lines
are list prices read from the same AWS pricing pages on the same date (see
**Sources**). One month = **730 hours**.

## Assumptions

| Assumption | Value |
|---|---|
| Active days / month | 30 |
| Messages / user / day | 10 |
| Input tokens / message | ~1,500 (prompt + RAG-retrieved context) |
| Output tokens / message | ~500 |
| RAG queries | 1 per 2 messages (50%) — served by **self-hosted Qdrant on EC2**, so no per-query cloud fee |
| File uploads | 5 MB / user / month, **12-month retention** |
| LLM | **Claude Haiku 4.5 on Amazon Bedrock**, Standard on-demand tier, `eu-west-1` |
| Embeddings | Bedrock Titan Text Embeddings v2 — negligible (~$1–3/mo even at 1000 MAU), folded into Bedrock |

**Architecture scales with demand** (mirroring the Q2 Dev→Stage→Prod progression),
not a full prod fleet at every tier:

- **10 MAU** — single `t3.xlarge`, everything in containers (Postgres, MinIO, Qdrant, MCPHub, Hayhooks), **Caddy** for TLS. No RDS/S3/ALB.
- **100 MAU** — single `t3.2xlarge`, Postgres/Qdrant containers on persisted EBS, files on **S3**, still Caddy.
- **1000 MAU** — full Q2 **prod fleet**: `c6i.2xlarge` (app + Hayhooks) behind an **ALB**, `r6i.large` (Qdrant), `t3.medium` (Casdoor + MCPHub), **RDS PostgreSQL Multi-AZ**, S3, CloudWatch.

**Bedrock math @ 1000 MAU (the number that drives everything):**
`1000 users × 10 msg × 30 days = 300,000 messages/mo`.
Input `300k × 1,500 = 450 MTok × $1.00 = $450`. Output `300k × 500 = 150 MTok × $5.00 = $750`. **Total $1,200/mo.**
(The screenshot price of $1.00 / $5.00 is **higher** than my earlier $0.80 / $4.00 estimate, so the real bill is ~25% bigger than first drafted — Bedrock dominates even more than expected.)

## Top 3 cost drivers at 1000 MAU

| # | Driver | Monthly | Share | Behaviour |
|---|---|---|---|---|
| 1 | **Bedrock (LLM tokens)** | $1,200 | **~65%** | **Variable** — scales linearly with every message |
| 2 | **EC2 compute fleet** | $414 | ~22% | Mostly **fixed** (step function as you add instances) |
| 3 | **RDS Multi-AZ** | $133 | ~7% | Fixed |

The remaining ~6% is EBS, S3, ALB and networking. The headline: **two-thirds of the
bill is LLM inference, and it is the only line that grows with usage.** Fixed infra
is a rounding error once you are at scale.

## Cost-cutting levers (each ≥15%, with the trade-off)

**Lever 1 — Self-host the LLM on a GPU EC2 instead of Bedrock.**
Run a quantized open model (e.g. Qwen3-30B or Llama-3.x) on a `g4dn.xlarge` (T4 16 GB,
~$0.59/h `eu-west-1` ≈ **$430/mo**) and drop Bedrock entirely. Saving:
`$1,200 → $430 = −$770/mo (~42% of total)`. Total falls to **~$1,080/mo**.
*Trade-off:* lower answer quality vs. Claude, 2–3× latency, no managed SLA, a single
T4 caps throughput so peaky load needs a second GPU or autoscaling, and the ops/MLOps
burden (model updates, GPU patching) moves to Daveo.

**Lever 2 — Bedrock prompt caching + response-length control.** *(recommended)*
The screenshot shows Haiku 4.5 **cache-read at $0.10/MTok** (vs $1.00 input) and a
5-min cache write at $1.25. The system prompt + reused RAG context (~60% of input
tokens) become cache hits, and capping output verbosity (500 → ~300 tokens) cuts the
expensive `$5/MTok` output line:
- Input `$450 → ~$200` (caching),
- Output `$750 → ~$450` (shorter responses).
Bedrock `$1,200 → ~$650 = −$550/mo (~30% of total)`. Total falls to **~$1,300/mo**.
*Trade-off:* the 5-min cache TTL means low-traffic users miss the cache; shorter
answers can truncate long proposal drafts; needs prompt-engineering + an eval harness
to confirm quality holds. **Lower risk than Lever 1, so this is the one I'd ship first.**

## Unit economics

| Scale | Total / mo | **Cost / MAU / mo** |
|---|---|---|
| 10 MAU | $159 | **$15.90** |
| 100 MAU | $418 | **$4.18** |
| 1000 MAU | $1,846 | **$1.85** |

The per-user cost falls steeply (10× cheaper from 10→1000 MAU) as the fixed EC2/RDS/ALB
base is amortised across more users. But it does **not** keep falling: the **marginal
cost of one extra user is essentially the Bedrock spend, ≈ $1.20/MAU** (plus a few
cents of storage/transfer). That is the floor.

The curve "knee" is around **~500 MAU**: below it, fixed infrastructure dominates and
adding users is highly accretive; above it, the bill is **token-dominated** and
per-user cost asymptotes toward ~$1.50–1.85. Past that point, scaling buys little
extra efficiency — the only way down is the two levers above (cheaper inference or
fewer tokens).

## Pricing recommendation

For Daveo this is an **internal** tool, so "pricing" means an internal chargeback that
also makes the business case defensible:

- **Charge ~$8 / user / month** as an internal seat price. That is ~2× the fully-loaded
  cost at the 100-MAU pilot ($4.18) — enough margin to fund support and the platform
  team — and becomes very comfortable (~4× cost) at 1000 MAU ($1.85). Against the value
  (tender cycle cut from 4–6 h to <45 min across ~300 consultants), $8/seat is trivially
  justified.
- **Bundle a fair-use token cap into every seat.** Because Bedrock is ~65% of cost and
  scales *linearly per message*, a handful of power users can erase the margin. Cap the
  included tokens and move heavy users to a metered/overage tier.
- **If this were sold externally** as SaaS, price ~$15–20/user/mo at the 100-MAU stage
  (cost + support + sales & marketing + margin), with the cost basis improving as scale
  drives per-user cost toward $1.85.
